# 网易新闻 Push 数据分析 Agent 系统 — 设计规范文档 (DESIGN.md)

---

## 一、设计总则

### 1.1 产品定位

本系统是网易新闻 Push 推送业务的数据分析 Agent 平台，面向**数据分析师**和**管理层**，提供从「异常发现 → 归因定位 → 策略建议 → 执行闭环」的全链路智能化分析体验。

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| **高效决策** | 数据密度高但不杂乱，关键信息 3 秒内可定位 |
| **状态可视** | 18 种业务状态（S01-S18）在任何页面都有明确视觉表达 |
| **操作流畅** | 核心路径（告警→归因→策略）页面跳转不超过 3 步 |
| **降低认知负荷** | 复杂归因链路的可视化表达，让非技术用户也能理解 |

### 1.3 设计原则

1. **数据优先** — 装饰元素最小化，空间让位给数据
2. **渐进呈现** — 先总览再细节，先结论再证据
3. **状态驱动** — 所有 UI 元素都有明确的「状态色」对应
4. **置信度透明** — Agent 输出的结论必须标注置信度，不确定时不说确定话

## 五、通用组件规范

### 5.1 状态标签 (Status Tag)

所有 Agent 状态码（S01-S18）使用统一 Tag 组件：

| 状态类型 | 颜色 | 示例 |
|---------|------|------|
| 正常/完成/已闭环 | 绿色 | `已闭环 S15` |
| 进行中/巡检中/归因中 | 蓝色（带脉冲动画） | `归因中 S07` |
| 告警/关注 | 橙色 | `告警 S04` |
| 严重/超期/未达标 | 红色 | `严重告警 S05` |
| 待处理/待审核 | 默认灰 | `待审核 S11` |
| 异常/数据延迟 | 灰色 + ⚠️ 图标 | `数据延迟 S06` |

### 5.2 置信度仪表盘

用于归因结论的置信度展示：

- 半圆环仪表（Gauge），范围 0-100
- 颜色分段：0-59 红色 / 60-79 黄色 / 80-100 绿色
- 中心显示数字
- 低置信度（<60%）旁显示 ⚠️「建议人工复核」

### 5.3 趋势指示器

用于数据变化的方向和幅度展示：

```
↑ 12.5%  (绿色，表示上升，通常正面)
↓ 37.0%  (红色，表示下降，通常负面)
→ 0.3%   (灰色，变化不显著)
```

- 箭头方向 + 百分比 + 颜色
- 变化 < 1% 时用 → 平箭头（灰色）

### 5.4 异常脉冲动画

新告警到达时的视觉反馈：

```css
@keyframes alertPulse {
  0%   { box-shadow: 0 0 0 0 rgba(245, 63, 63, 0.4); }
  70%  { box-shadow: 0 0 0 8px rgba(245, 63, 63, 0); }
  100% { box-shadow: 0 0 0 0 rgba(245, 63, 63, 0); }
}
```

### 5.5 加载与空态

| 场景 | 处理 |
|------|------|
| 数据加载中 | Skeleton 骨架屏（卡片/表格/图表各有对应骨架） |
| 列表无数据 | 空态插画 + 「暂无数据」+ 相关操作引导 |
| 归因进行中 | 进度指示器 + 步骤动画 + 预估剩余时间 |
| 数据源延迟 | 黄色横幅：「数据更新延迟，当前为 T+1 数据」+ 预计恢复时间 |
| 网络错误 | 红色横幅 + 重试按钮 |

- 点击展开通知面板（Dropdown）

## 六、响应式策略

本期原型以 **桌面端（1440px+）** 为主，最低支持 1280px。

| 断点 | 布局调整 |
|------|---------|
| ≥ 1440px | 6 列数据卡片 + 2 列图表 |
| 1280px – 1439px | 4 列数据卡片 + 2 列图表 |
| < 1280px | 侧边栏收缩为图标模式，卡片 3 列 |
| < 1024px | 本期不做适配（提示用户使用更大屏幕） |

## 七、交互规范补充

### 7.1 路由跳转规则

```
数据看板 ──点击告警行──→ 异常详情 ──触发归因──→ 归因分析 ──生成策略──→ 策略建议
    ↑                       ↑                    ↑               │
    └───────────────────────┴────────────────────┴─── 处理记录 ←──┘
```

### 7.2 数据刷新策略

| 页面 | 刷新策略 |
|------|---------|
| 数据看板 | 每 5 分钟自动轮询，有异常立即推送（WebSocket） |
| 异常详情 | 打开时获取最新数据，每 1 分钟轮询 |
| 归因分析 | 进行中每 10 秒轮询进度；完成即停止 |
| 策略建议 | 打开时加载，检查点触发时推送更新 |
| 处理记录 | 手动刷新或切换筛选条件时重新加载 |

### 7.3 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl/Cmd + K` | 全局搜索（告警、归因报告、策略建议） |
| `R` | 手动刷新当前页面 |
| `Esc` | 关闭当前弹窗/抽屉 |
